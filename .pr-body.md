## Problem

Clicking **View** on "My Admin Entry Key" in the Security Settings panel threw `Error: invalid_password`. The old `viewCurrentKey()` called `/api/security/admin-key/view` which required validating the login password via `bcrypt.compare()`. This failed for dev admin users because the dev admin authenticates via a plaintext password check, not a stored `passwordHash`.

## Solution

Replaced the server-round-trip approach with direct localStorage reads/writes:

### Bug Fix
- `viewMyKey()` — Reads key from localStorage and displays it in an inline monospace reveal box (auto-hides after 5 seconds). No password prompt needed.
- `saveMyKey()` — Stores to localStorage first (instant), then syncs to server in background.
- `clearKey()` — Removes from localStorage first, then syncs deletion to server.

### UI Improvements
- Added `getCurrentUser()` helper to decode JWT for per-user key storage.
- Added inline reveal box (my-key-reveal-box) for displaying keys.
- User Management cards now show **View Key** and **Delete Key** buttons when a key exists.
- **KEY ACTIVE / NO KEY** badges driven by localStorage state.
- Simplified `initPasswordToggle()` to delegate to `initKeyToggle()` (eliminates duplicate listeners).

## Testing
- All **88 tests pass** with no regressions.
- Manual verification: Save, View, Clear cycle works without errors.
